## 시작하기

약 한달간의 프로젝트를 시작하게 되었습니다. FE 3분과 BE 3분으로 팀이 편성되어서 약 1주일간 아이디어 기획과 각종 컨벤션을 작성하는 것에 많은 시간을 추가했습니다. 우선 가장 어려웠던 점은 어떤 아이디어를 구현하고 1달의 기간 동안 우리 수준에 맞게 어디까지 실현할 수 있는지 범위를 잡는 것이 어려웠습니다. 

## 주제 정하기

여러 아이디어를 제안하는 과정에서 최종적으로 선택한 주제는 반려동물이었습니다. 반려동물과 관련된 주제를 선택한 이유는 반려동물이 있는 집이 많고 강아지, 고양이 뿐만 아니라 특색 있는 반려 동물과 함께 살고 있는 분들도 많다는 것을 알게 되었습니다. 

그렇다면, 반려동물과 관련된 프로젝트로 어떤 것을 만들어야 할까? 하였을 때, 우선적으로 쇼핑몰은 경쟁력이 없고 세부 아이템 혹은 제품을 컨택하는 것에서 어려움이 있다고 생각하였습니다.  그래서 자체적으로 생태계를 만들수 있는 웹 어플리케이션을 만들기로 하였습니다. 그래서 반려동물이 있는 사람들을 유입하기 위해서 저희가 선택한 아이디어는 SNS 기능이었습니다. SNS에서 초기 유저 유입이 어려운 점은 이미 인스타그램, 페이스북, 트위트, 틱톡과 같은 특색있는 SNS가 있는 상황에서 매우 초보적인 기술을 가지고 있는 저희의 앱을 사용할 이유를 제공해야 했습니다. 이때 당근마켓의 아이디어를 차용했습니다. 한편으로는 SNS 기능을 제공하면서, 추가적으로 반려동물이 있는 사람들 끼리 함께 산책할 수 있는 기능을 추가하여 특색있는 장점을 만들기로 결정하였습니다. 그래서 결정된 부분은 크게 두 기능이었습니다.

1. 반려동물 사진, 영상 등을 업로드 하는 SNS 기능
2. 반려동물과 함께 사는 사람들이 함께 모여 산책할 수 있는 커뮤니케이션 역할

## 기능 추가 및 덜어내기

메인 아이디어를 바탕으로 어떤 기능이 필요한지 회의를 진행하게 되었습니다. 이를 통해서 결정한 기능은 다음과 같습니다. 

### 유저 시나리오

- 유저 회원가입 → 펫 등록창으로 이동 → 펫 등록 및 인증 → 게시글 작성 권한 부여
    - 상세 거주지역 말고 동, 구까지만 정보를 받기
- 견주 프로필
    - 유저 회원가입 정보 -이름, 나이, 성별, 거주지역
    - 다른 유저들이 보는 것은 사는 지역, 닉네임, 펫에 대한 정보
    - 견주 사진은 자신이 셀렉
    - OAuth 가능 - kakao

## 펫등록

- 펫 사진, 이름, 나이, 성별, 종, 성격, 한줄 소개, 산책 가능 여부

## 페이지

- 페이지는 사진 게시 페이지와 산책 페이지로 나눈다.
- 게시물 등록과 산책 등록 모두 반려동물 프로필이 아닌 **견주 프로필로 등록한다.**
1. **게시물**
    1. 팔로우 기능
    2. 좋아요 기능 
    3. 댓글 기능 
    4. 댓글 좋아요 기능 
    5. 공유 기능 
    6. 태그를 통한 검색 기능, 태그는 자유롭게 작성 가능, 태그를 적을 수 있는 글자 수만 제한 
        - 부가기능
            1. **삭제한 게시물 복원** 
            
2. **산책**
    1. 본인의 거주 지역만 보이게 하기 
    2. 산책 메이트 이후에 대한 당근 마켓 매너 온도를 차용해서 평가 작성(리뷰 작성)
    3. 산책 메이트 작성시 반려동물을 서로 알아 볼 수 있는 아이템을 필수록 작성하기 (옷, 장신구, 손잡이 베지, 목줄, 등등), 색을 통한 확인(견주)
    4. 게시물 등록시 인원 선택 기능
    5. 펫 등록 창에 작성한 동물의 성격 / 종 / 나이 / 짧은 소개 미리보여 주기
    6. 모집 중, 모집 마감
    7. 인원 모집 마감 기능 (마감 표시 - 흑색 등등)
        - 부가기능
        1. **마감 기한 등록 기능**
        2. **지도 기능**
        
3. **마이페이지 (견주를 위한 마이페이지)**
- 유저 정보 수정 기능
- 팔로워 팔로잉
- 본인이 올린 게시물 확인 (게시글, 산책 올린 글) → 위치 구분은 필요함
- 견주 본인인지에 따라 반려 동물 수정 버튼 유무
- 반려 동물 추가 삭제, 반려 동물 정보 수정
- 반려 동물 구성원 보는 컴포넌트

1. **검색**
- 페이지를 따로 만들기
- 사용자 이름을 통한 견주 검색 (그냥 검색)
    - 추천 기능으로 적은 키워드가 포함된 견주 아이디 리스트를 보여주어야 한다.
- 태그 이름을 통한 검색
    - 관련 태그 유저에게 제공
- 검색 결과 유저 페이지는 마이페이지에서의 edit 권한을 제외

1. **채팅 기능 (부가 기능)**
    1. 웹 소켓
    2. 산책 페이
        - 부가기능
        1. **단체 채팅**

## 최종정리
